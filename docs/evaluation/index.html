If you can't measure it, you can't improve it. Learn to evaluate your SLM rigorously before deploying.

Core Metrics

MetricWhat It MeasuresWhen to UseGood Score
Accuracy% predictions exactly correctBalanced classes>85%
PrecisionOf positive predictions, % actually positiveFalse positives costly>80%
RecallOf actual positives, % foundFalse negatives costly>80%
F1 ScoreHarmonic mean of precision & recallImbalanced classes>75%
Exact MatchOutput matches expected exactlyStructured output>60%
PerplexityModel confidence on held-out dataLanguage modellingLower is better
python
from sklearn.metrics import accuracy_score, f1_score, classification_report

y_true = ['J18.9', 'I10', 'E11.9', 'J18.9']
y_pred = ['J18.9', 'I10', 'E11.65', 'J18.1']

print(f'Accuracy: {accuracy_score(y_true, y_pred):.1%}')
print(f'F1 macro: {f1_score(y_true, y_pred, average="macro"):.1%}')
print(classification_report(y_true, y_pred))

Hallucination Testing

🚨
Hallucination is your #1 production risk

A medical coding model that confidently assigns the wrong ICD-10 code can cause billing fraud. Always test before deploying in high-stakes domains.

Three types of hallucination to test:

  • Fabrication — inventing codes, citations, or facts that don't exist
  • Substitution — substituting a similar but wrong entity (I21.9 instead of I21.0)
  • Contradiction — output contradicts information in the input context
python
import json

VALID_CODES = set(json.load(open('icd10_codes.json')))

def check_hallucination(model_output):
    codes = [c.strip() for c in model_output.split(',')]
    invalid = [c for c in codes if c not in VALID_CODES]
    return {
        'hallucinated': invalid,
        'rate': len(invalid) / len(codes) if codes else 0
    }

Building a Benchmark Suite

1
Separate 15–20% of data BEFORE training
Never train on test data. Create a fixed test set on day 1 and never change it.
2
Include edge cases deliberately
Add the hardest, most ambiguous examples you know about. These are your canaries.
3
Run after every checkpoint
Log to Weights & Biases or MLflow. Watch the learning curve — catch overfitting early.
4
Compare against baselines
Always compare: (a) base model zero-shot, (b) GPT-4 zero-shot. Know what you're beating.
