Troubleshooting

Stuck? Find solutions to the most common SLM development errors here.

50+ errors covered

CUDA & GPU Errors

CUDA out of memory (OOM)
Reduce batch size: per_device_train_batch_size=1
Enable gradient checkpointing: gradient_checkpointing=True
Use QLoRA instead of LoRA: adds 4-bit quantization to the base model
Clear cache: torch.cuda.empty_cache() between training runs
Increase gradient accumulation: gradient_accumulation_steps=8 to simulate larger batches
Reduce sequence length: max_seq_length=512 instead of 2048
RuntimeError: CUDA error: device-side assert triggered
Usually caused by invalid token IDs in your dataset. Set CUDA_LAUNCH_BLOCKING=1 to get the real error. Check that your tokenizer's vocab size matches the model. Verify labels are within [0, vocab_size).
CUDA not available (torch.cuda.is_available() returns False)
1. Check NVIDIA driver: nvidia-smi
2. Check CUDA version: nvcc --version
3. Reinstall PyTorch with matching CUDA: pip install torch --index-url https://download.pytorch.org/whl/cu121
4. In Colab: Runtime → Change runtime type → T4 GPU

Training Issues

Training loss not decreasing / stuck at same value
Common causes:
1. Learning rate too high or too low — try 2e-4 for QLoRA
2. Dataset not formatted correctly — check instruction/output columns exist
3. All outputs are the same in your dataset — add diversity
4. Model is already fine-tuned on this data — use a base model, not instruct
Training loss is NaN
NaN loss means the model exploded. Fix: lower your learning rate (try 1e-5), add max_grad_norm=0.3, use bfloat16 instead of float16 (more stable), reduce batch size.

Ollama Issues

Ollama: “model not found”
Run ollama pull llama3.2:1b first. Check available models with ollama list. If using a custom model, verify the Modelfile path is correct.
Ollama very slow on CPU
CPU inference is expected to be slow (10–30 tokens/sec). Use Q4_K_M quantization for best CPU speed. On Apple Silicon, Ollama uses Metal automatically — check with ollama ps. For NVIDIA GPU, ensure CUDA drivers are installed before Ollama.

vLLM Issues

vLLM: ValueError: The model's max seq len is larger than the maximum
Add --max-model-len 4096 (or lower) to your vllm command. This overrides the model's config to fit in your VRAM.
vLLM: Cannot import name ‘LLM’
Upgrade vLLM: pip install --upgrade vllm. If on Python 3.12+, check vLLM supports your Python version.

HuggingFace Issues

OSError: We couldn’t connect to HuggingFace Hub
1. Check internet connection
2. Login: huggingface-cli login
3. For gated models (Llama), accept the license at huggingface.co/meta-llama
4. Set HUGGING_FACE_HUB_TOKEN environment variable
