🔋 Fine-Tuning — QLoRA

QLoRA Fine-Tuning Guide

Fine-tune a 7B model on 6GB VRAM. The most important technique in practical SLM development.

⏱ 20 min readIntermediate

How QLoRA Works

QLoRA = Quantize the base model to 4-bit NF4 precision (reduces memory 4x) + apply LoRA adapters on top (only train the small adapter matrices). The key insight: the frozen 4-bit weights are only dequantized when needed for computation, keeping memory low while preserving training quality.

✅
Why QLoRA changed everything

Before QLoRA (2023): fine-tuning a 7B model required 80GB VRAM (an A100). After QLoRA: the same fine-tune runs on a consumer RTX 3090 (24GB) or even a T4 (16GB) for smaller models. This democratised SLM development.

Key Hyperparameters

ParameterRecommendedEffect
r (LoRA rank)16 (simple tasks), 32 (complex)Higher = more capacity, more VRAM
lora_alpha2x rank (32 or 64)Scaling factor for LoRA updates
lora_dropout0.05Regularization. 0.0 with Unsloth.
learning_rate2e-4Higher for small datasets, lower for large
num_train_epochs3More epochs risks overfitting
batch_size4 per deviceReduce if OOM
grad_accum_steps4Effective batch = batch_size x accum
max_seq_length1024–2048Longer = more VRAM

Complete Training Code

python
# Install: pip install unsloth trl peft transformers datasets bitsandbytes
from unsloth import FastLanguageModel
from trl import SFTTrainer
from transformers import TrainingArguments
from datasets import Dataset
import torch

# 1. Load model in 4-bit
model, tokenizer = FastLanguageModel.from_pretrained(
    model_name='unsloth/Llama-3.2-1B-Instruct',  # or 3B, Phi-4-mini, Qwen2.5-3B
    max_seq_length=1024,
    load_in_4bit=True,
    dtype=torch.bfloat16,
)

# 2. Add LoRA adapters
model = FastLanguageModel.get_peft_model(
    model, r=16, lora_alpha=32, lora_dropout=0.05,
    target_modules=['q_proj','k_proj','v_proj','o_proj','gate_proj','up_proj','down_proj'],
)
model.print_trainable_parameters()

# 3. Prepare dataset (instruction-following format)
def format(ex):
    return {'text': f'### Instruction:\n{ex["instruction"]}\n### Input:\n{ex["input"]}\n### Response:\n{ex["output"]}'}

data = [{'instruction':'...','input':'...','output':'...'}, ...]  # your data
dataset = Dataset.from_list(data).map(format)
train_ds, eval_ds = dataset.train_test_split(0.15).values()

# 4. Train
trainer = SFTTrainer(
    model=model, tokenizer=tokenizer,
    train_dataset=train_ds, eval_dataset=eval_ds,
    dataset_text_field='text', max_seq_length=1024,
    args=TrainingArguments(
        output_dir='./output',
        num_train_epochs=3,
        per_device_train_batch_size=4,
        gradient_accumulation_steps=4,
        learning_rate=2e-4,
        warmup_ratio=0.05,
        lr_scheduler_type='cosine',
        optim='paged_adamw_8bit',
        bf16=True, logging_steps=10,
        eval_strategy='steps', eval_steps=50,
        save_strategy='steps', save_steps=50,
        load_best_model_at_end=True,
    ),
)
trainer.train()

# 5. Save
model.save_pretrained('my-slm')
tokenizer.save_pretrained('my-slm')

# 6. Export to GGUF for Ollama
model.save_pretrained_gguf('my-slm-gguf', tokenizer, quantization_method='q4_k_m')
