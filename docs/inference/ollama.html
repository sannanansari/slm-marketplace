🦙 Ollama

Run Any SLM Locally with Ollama

Ollama is the easiest way to run language models on your own machine. One command installs it, one command downloads and runs a model. This guide covers installation, the REST API, Modelfiles, calling from Python/JavaScript, and serving your own fine-tuned GGUF model.

⏱ 25 min read 💻 Mac / Linux / Windows 🎯 Beginner-friendly

Install Ollama

bash
# Download the macOS app from ollama.com
# Or install via Homebrew:
brew install ollama

# Start the Ollama server (runs in background)
ollama serve &

# Verify installation
ollama --version
Apple Silicon: Ollama natively uses Metal GPU acceleration on M1/M2/M3/M4 Macs. No configuration needed — it's automatic.
bash
# One-line install (official script)
curl -fsSL https://ollama.com/install.sh | sh

# The installer:
# - Detects your GPU (NVIDIA/AMD/CPU)
# - Installs CUDA libraries if needed
# - Creates systemd service (auto-start on boot)
# - Adds 'ollama' to PATH

# Check it's running
systemctl status ollama

# Or start manually
ollama serve

Run Your First Model

bash

ollama list

ollama rm llama3.2:1b

# Show model info
ollama show llama3.2:1b

Ollama REST API

Ollama runs a local REST API on port 11434. You can call it from any language.

bash
# Generate (single turn)
curl http://localhost:11434/api/generate \
  -d '{
    "model": "llama3.2:1b",
    "prompt": "What is a transformer model?",
    "stream": false
  }'

# Chat (multi-turn with history)
curl http://localhost:11434/api/chat \
  -d '{
    "model": "llama3.2:1b",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant."},
      {"role": "user",   "content": "What is fine-tuning?"}
    ],
    "stream": false
  }'

# Streaming response (real-time token output)
curl http://localhost:11434/api/chat \
  -d '{
    "model": "llama3.2:1b",
    "messages": [{"role": "user", "content": "Count to 5"}],
    "stream": true
  }'

# List available models
curl http://localhost:11434/api/tags

# Generate embeddings
curl http://localhost:11434/api/embed \
  -d '{
    "model": "nomic-embed-text",
    "input": "Text to embed for RAG"
  }'

javascript
// npm install ollama
import ollama from 'ollama';

// Simple chat
async function ask(question) {
  const response = await ollama.chat({
    model: 'llama3.2:1b',
    messages: [{ role: 'user', content: question }]
  });
  return response.message.content;
}

// Streaming
async function streamChat(question) {
  const stream = await ollama.chat({
    model: 'llama3.2:1b',
    messages: [{ role: 'user', content: question }],
    stream: true
  });

  for await (const chunk of stream) {
    process.stdout.write(chunk.message.content);
  }
  console.log();
}

// Full chatbot with history
class OllamaChatbot {
  constructor(model = 'llama3.2:1b', systemPrompt = 'You are helpful.') {
    this.model = model;
    this.messages = [{ role: 'system', content: systemPrompt }];
  }

  async send(userMessage) {
    this.messages.push({ role: 'user', content: userMessage });

    const response = await ollama.chat({
      model: this.model,
      messages: this.messages
    });

    const reply = response.message.content;
    this.messages.push({ role: 'assistant', content: reply });
    return reply;
  }
}

Modelfiles — Customize Any Model

A Modelfile is like a Dockerfile for AI models. It lets you customize the system prompt, temperature, context length, and more — then bake it into a named model you can share.

bash
# Modelfile — create a domain-specific assistant # Base model to customize FROM llama3.2:1b # System prompt — defines the assistant's persona and behavior SYSTEM """ You are LegalEagle, an expert legal assistant specializing in contract analysis. Your role: - Identify key clauses and their implications - Flag unusual or risky terms - Explain legal language in plain English - Always note when users should consult a licensed attorney Formatting: - Use bullet points for clause analysis - Bold key terms using **bold** - Always include a "Risk Level: Low/Medium/High" rating You do NOT provide legal advice. You provide legal information. """ # Model parameters PARAMETER temperature 0.3 # Low temp for factual, consistent answers PARAMETER top_p 0.9 PARAMETER num_ctx 4096 # Context window PARAMETER repeat_penalty 1.1 # Reduce repetition PARAMETER num_predict 500 # Max tokens per response # Optional: custom stop tokens # PARAMETER stop "<|end|>"
bash
# Create the custom model from the Modelfile ollama create legaleagle -f ./Modelfile # Run it ollama run legaleagle # >>> Analyze this clause: "Either party may terminate this agreement # >>> with 30 days written notice for any reason." # Output: # **Termination Clause Analysis** # • Type: At-will termination with notice # • Notice period: 30 days (written) # • Bilateral: Both parties have equal termination rights # **Risk Level: Low** — Standard commercial clause... # Push to Ollama registry (create account at ollama.com) ollama push your-username/legaleagle

Load Your Own Fine-Tuned GGUF Model

After fine-tuning and exporting to GGUF, you can load it directly into Ollama:

bash
# Method 1: Load GGUF directly from file path cat > MyModelfile << 'EOF' FROM ./my-model-q4_k_m.gguf SYSTEM "You are a domain expert assistant trained on specialized data." PARAMETER temperature 0.4 PARAMETER num_ctx 4096 EOF ollama create my-domain-model -f MyModelfile ollama run my-domain-model # Method 2: Load directly from HuggingFace (Ollama 0.6+) ollama run hf.co/your-username/your-model-GGUF:Q4_K_M # This downloads the Q4_K_M GGUF from your HuggingFace repo # and creates a local Ollama model automatically

Production Configuration

bash
# Environment variables for production Ollama # Allow external connections (default: localhost only) OLLAMA_HOST=0.0.0.0:11434 # Number of models kept loaded in memory (default: 1) OLLAMA_MAX_LOADED_MODELS=3 # Keep models loaded in memory for N minutes after last use OLLAMA_KEEP_ALIVE=30m # Use GPU (auto-detected, but can force CPU) OLLAMA_GPU_OVERHEAD=0 # Log level OLLAMA_DEBUG=false # Set via systemd (Linux): sudo systemctl edit ollama # Add: # [Service] # Environment="OLLAMA_HOST=0.0.0.0:11434" # Environment="OLLAMA_MAX_LOADED_MODELS=3" # Restart sudo systemctl restart ollama
python
# production_ollama.py — production wrapper with error handling import ollama import time import logging from typing import Optional logger = logging.getLogger(__name__) class OllamaClient: """Production wrapper around Ollama with retry logic.""" def __init__(self, model: str, host: str = "http://localhost:11434"): self.model = model self.client = ollama.Client(host=host) self._verify_model() def _verify_model(self): """Check model is available before first request.""" models = [m["name"] for m in self.client.list()["models"]] if not any(self.model in m for m in models): logger.warning(f"Model '{self.model}' not found. Pulling...") self.client.pull(self.model) def chat( self, messages: list[dict], temperature: float = 0.7, max_tokens: int = 500, retries: int = 3 ) -> Optional[str]: """Send chat request with retry on failure.""" for attempt in range(retries): try: response = self.client.chat( model=self.model, messages=messages, options={ "temperature": temperature, "num_predict": max_tokens, "repeat_penalty": 1.1 } ) return response["message"]["content"] except ollama.ResponseError as e: logger.error(f"Ollama error (attempt {attempt+1}): {e}") if attempt < retries - 1: time.sleep(2 ** attempt) # exponential backoff except Exception as e: logger.error(f"Unexpected error: {e}") raise return None def generate_with_timing(self, prompt: str) -> dict: """Generate and return response with performance metrics.""" start = time.perf_counter() response = self.client.generate( model=self.model, prompt=prompt, stream=False ) elapsed = time.perf_counter() - start tokens = response.get("eval_count", 0) tps = tokens / elapsed if elapsed > 0 else 0 return { "text": response["response"], "tokens": tokens, "elapsed_s": round(elapsed, 2), "tokens_per_sec": round(tps, 1) } # Usage: # client = OllamaClient("llama3.2:1b") # result = client.generate_with_timing("What is deep learning?") # print(f"Answer: {result['text']}") # print(f"Speed: {result['tokens_per_sec']} tok/s")

Troubleshooting

ErrorCauseFix
connection refused Ollama server not running Run ollama serve or systemctl start ollama
model not found Model not pulled yet Run ollama pull model-name first
Very slow (CPU) GPU not detected NVIDIA: install nvidia-cuda-toolkit. AMD: ROCm support via env var.
out of memory Model too large for RAM/VRAM Use smaller model or lower quant (Q4 instead of Q8)
Port 11434 in use Another process using port Set OLLAMA_HOST=0.0.0.0:11435
Model repeats itself Repetition penalty too low Add PARAMETER repeat_penalty 1.15 to Modelfile

/api/generate is a simple completion endpoint — you provide a raw prompt string and get a completion. /api/chat takes a messages array with roles (system/user/assistant) and handles conversation formatting automatically. Use /api/chat for anything conversational.
