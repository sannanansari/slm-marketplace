⚡ Inference — vLLM

vLLM — Production Serving

10–25x higher throughput than naive inference. OpenAI-compatible API. The production standard for SLM serving.

⏱ 10 min readAdvanced

Install

bash
# Requires NVIDIA GPU with CUDA 12.1+
pip install vllm

# Verify
python -c 'import vllm; print(vllm.__version__)'

Start the API Server

bash
# Basic server (OpenAI-compatible)
python -m vllm.entrypoints.openai.api_server \
  --model meta-llama/Llama-3.2-1B-Instruct \
  --host 0.0.0.0 \
  --port 8000 \
  --max-model-len 4096

# With your fine-tuned model from HuggingFace
python -m vllm.entrypoints.openai.api_server \
  --model your-username/YourFineTunedModel \
  --max-model-len 4096 \
  --gpu-memory-utilization 0.90

# Multi-GPU (for 7B+ models)
python -m vllm.entrypoints.openai.api_server \
  --model meta-llama/Llama-3.1-8B-Instruct \
  --tensor-parallel-size 2 \
  --max-model-len 8192

Query the API

python
from openai import OpenAI

# vLLM uses the same client as OpenAI
client = OpenAI(base_url='http://localhost:8000/v1', api_key='not-needed')

# Chat completion
response = client.chat.completions.create(
    model='your-username/YourFineTunedModel',
    messages=[
        {'role': 'system', 'content': 'You are a medical coding expert.'},
        {'role': 'user',   'content': 'Patient with acute chest pain...'},
    ],
    max_tokens=256,
    temperature=0.1,
)
print(response.choices[0].message.content)

# Batch inference (much faster than one-by-one)
import asyncio
from openai import AsyncOpenAI

async def batch_predict(notes):
    aclient = AsyncOpenAI(base_url='http://localhost:8000/v1', api_key='-')
    tasks = [aclient.chat.completions.create(
        model='your-model', messages=[{'role':'user','content':n}], max_tokens=128
    ) for n in notes]
    return await asyncio.gather(*tasks)

results = asyncio.run(batch_predict(['note 1...', 'note 2...', 'note 3...']))

Performance Tuning

ParameterDefaultRecommendationEffect
--gpu-memory-utilization0.900.85–0.95More VRAM = more KV cache = higher throughput
--max-num-batched-tokensauto4096–8192Larger batches = higher GPU utilization
--quantizationnoneawq or fp82x memory savings, ~5% quality loss
--enable-prefix-cachingoffonCache repeated system prompts

# Production-optimised command
python -m vllm.entrypoints.openai.api_server \
