🗺️ 6-Month Roadmap

Zero to SLM Engineer in 6 Months

A realistic, day-by-day curriculum from absolute beginner to published AI engineer. Every week has learning objectives, exercises, a mini-project, and free resources.

⏱ 1–2 hours per day 🆓 100% free resources 💻 Hands-on from Day 1 🏆 Publish by Month 6
How to use this roadmap: Follow the months in order. Each day is 1–2 hours. Skip days you already know. The mini-projects build on each other — by Month 6 you have a complete, published SLM.
Month 1 — Foundations & First Steps
Goal: Understand AI/ML fundamentals and run your first SLM locally
Python · Linux · Git · Math Basics · Run Ollama
Month 1 · Week 1
Python & Linux Fundamentals
Before you touch AI, you need the tools. Python is the language of AI. Linux is where models run in production. Git tracks your work. This week sets the foundation everything else builds on.
Day 1
Python Environment Setup
Install Python 3.11+. Learn pip, virtual environments, and Jupyter notebooks. Run your first Python script.
Day 2

Integers, floats, strings, booleans. if/else, for loops, while loops. List comprehensions.
Day 3
Python Basics II — Functions & Modules
def, args, kwargs, return values. Import standard library modules. Write reusable functions.
Day 4
Python Basics III — Files, JSON, Error Handling
Read/write files. Parse JSON. try/except. This is how you load training datasets.
Day 5
Linux Command Line
cd, ls, mkdir, cp, mv, rm, cat, grep, curl, wget, ps, kill. SSH basics. Why this matters for servers.
Day 6
Git & GitHub
init, add, commit, push, pull, branch, merge. Create a GitHub account. Push your first repo.
Day 7
Review + Practice
Solve 10 Python exercises on HackerRank. Commit every solution to GitHub. Set up VS Code with Python extension.

Month 3 — Fine-Tuning with LoRA, QLoRA & Unsloth
Goal: Fine-tune a real SLM on your domain dataset. Your first published model.

Month 3 · Week 9
LoRA & QLoRA — The Fine-Tuning Revolution
LoRA (Low-Rank Adaptation) is why you can fine-tune a 7B parameter model on a single consumer GPU. Understanding it is non-negotiable for SLM engineering.
Day 57
Why Full Fine-Tuning Is Impractical
VRAM requirements. A100 vs consumer GPU gap. Why fine-tuning GPT-3 costs $4.6M. The need for PEFT.
Day 58
LoRA Theory
Low-rank matrix decomposition. W = W₀ + AB where rank(AB) << rank(W). r and alpha hyperparameters. Target modules (q_proj, v_proj, k_proj).
Day 59
QLoRA — Quantized LoRA
Load base model in 4-bit (NF4 quantization). Keep LoRA adapters in fp16. Double quantization. The QLoRA paper explained.
Day 60
Unsloth — 2x Faster Fine-Tuning
What Unsloth optimizes (Flash Attention 2, custom CUDA kernels). Install. Supported models. Memory savings vs standard HuggingFace.
Day 61
SFTTrainer & TRL Library
HuggingFace TRL. SFTTrainer for supervised fine-tuning. TrainingArguments. DataCollatorForSeq2Seq. The training script structure.
Day 62
Hyperparameters for Fine-Tuning
LoRA r, alpha. Learning rate (2e-4 for LoRA). Epochs (1–3 typical). Batch size. Gradient accumulation steps. Warmup ratio.
Day 63
First Fine-Tune Run
Fine-tune Llama 3.2-1B on your domain dataset using Unsloth on Colab. Watch the loss curve. Spot overfitting. Save the adapter.

Quantization — Shrink Your Model Without Killing Quality
Day 85
What Is Quantization?
Reduce weight precision from fp32 to int8/int4. Less VRAM, faster inference, slight quality loss. The tradeoffs explained clearly.
Day 86
GGUF & llama.cpp

Day 87

Q2_K, Q4_K_M, Q5_K_M, Q8_0. Size vs quality comparison table. When each level is appropriate. Benchmark your model at each level.
Day 88
GPTQ Quantization
Post-training quantization with GPTQ. GPU-optimized. Use AutoGPTQ. Calibration dataset. When GPTQ beats GGUF.
Day 89
AWQ Quantization
Activation-aware Weight Quantization. Better quality than GPTQ at same bit width. AutoAWQ library. VRAM comparison.
Day 90
Choosing the Right Format
Decision tree: CPU-only → GGUF. GPU server → AWQ or GPTQ. Edge device → INT4 GGUF. Local Ollama → GGUF. Production API → AWQ+vLLM.
Day 91

Why RAG?
SLMs have limited context and stale knowledge. RAG adds a real-time knowledge base. Architecture: embed → store → retrieve → augment → generate.
Day 114
Embedding Models
sentence-transformers. all-MiniLM-L6-v2 vs BGE vs E5. Cosine similarity. How to embed your documents. Benchmark speed vs quality.

Day 116
Chroma Vector Database
Open-source, persistent, with metadata filtering. chromadb Python API. Collections. Persist to disk. Compare with FAISS.
Day 117
PGVector — Vectors in Postgres
pgvector extension for PostgreSQL (Supabase supports it!). Store embeddings alongside metadata. SQL + vector search in one query.
Day 118
Chunking Strategies
Fixed-size, sentence-based, semantic chunking. Overlap. Why chunk size matters more than model size. Recursive text splitting.
Day 119
RAG Pipeline End to End
Ingest PDFs → chunk → embed → store. At query time: embed query → retrieve top-k → build prompt → generate. Evaluate faithfulness.
