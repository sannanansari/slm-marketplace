Windows Setup for SLM Development

Set up Windows for SLM development using WSL2, native Python, CUDA, and Ollama. Both WSL2 and native Windows paths covered.

Windows and AI Development

Two paths: WSL2 (recommended — Linux environment inside Windows, full CUDA support) or native Windows (simpler, but some packages have limited support). For serious training, use WSL2. For inference with Ollama/LM Studio, native Windows is fine.

Path 1: WSL2 (Recommended for Training)

powershell
# In PowerShell (as Administrator) wsl --install # Restart your PC # After restart, open Ubuntu from Start menu # Then follow the Linux setup guide exactly: # docs/setup/linux.html
WSL2 + CUDA: Install NVIDIA drivers on Windows (not inside WSL2). WSL2 automatically passes through GPU access. Run nvidia-smi inside WSL2 to verify.

Path 2: Native Windows

powershell
# Install Python 3.11 from python.org (check "Add to PATH") # Open PowerShell python -m venv slm-env .\slm-env\Scripts\activate # PyTorch with CUDA 12.1 pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121 pip install transformers datasets accelerate pip install huggingface_hub # Ollama (native Windows installer) # Download from https://ollama.com/download # Then: ollama run llama3.2:1b
