Project Overview

Organizations lose 11% of contract value to leakage and 9% of annual revenue to poor contract management. The CLM software market exceeded $1.24 billion in 2025. Law firms cannot send client contracts to OpenAI — on-premise SLMs are the only option.

ContractSLM-3B uses Llama-3.2-3B (3B for better legal reasoning) fine-tuned on CUAD v1 — 510 real commercial contracts with 13,000+ expert annotations across 41 clause types, created by lawyers at The Atticus Project.

Dataset: CUAD v1

CUAD (Contract Understanding Atticus Dataset) is Apache 2.0 licensed — free for commercial use.

python
from datasets import load_dataset

# Downloads automatically, no credentials needed
cuad = load_dataset('theatticusproject/cuad-qa', trust_remote_code=True)
print(f'Training examples: {len(cuad["train"])}')
# 13,000+ labeled clause spans across 510 contracts

# The 41 clause types include:
# Limitation of Liability, Indemnification, IP Ownership,
# Non-Compete, Change of Control, Renewal Term, Governing Law,
# Termination, Audit Rights, Insurance... and 31 more

Training

bash
# 3B model needs more VRAM than 1B
# Recommended: Colab A100 (~3 hours) or RunPod RTX 3090 (~3 hours)
python build_dataset.py  # loads CUAD + generates synthetic risk examples
python train.py          # QLoRA rank 32 on Llama-3.2-3B-Instruct
python evaluate.py       # F1, risk accuracy, red flag recall
python push_to_hub.py --username YOUR_USERNAME --hf-token YOUR_TOKEN

Key Features

  • PDF/DOCX support — inference.py reads any file format via pdfplumber and python-docx
  • Long contract handling — automatically chunks contracts longer than 6,000 characters and merges results
  • REST API — python inference.py --serve starts an OpenAPI server with file upload support
  • Risk colour coding — CLI output shows 🟢 LOW, 🟡 MEDIUM, 🔴 HIGH, 🚨 CRITICAL per clause
