🌐 What is SLM Marketplace?

SLM Marketplace is a community-built discovery platform for Small Language Models — AI models under 7 billion parameters designed to run efficiently on standard hardware without expensive cloud APIs.

Think of it as the GitHub for SLMs: engineers publish their models, share benchmarks, and get feedback from the community. Businesses and developers discover production-ready models for their specific domain without building from scratch.

🔍 Discover

Search 200+ models across 8 categories. Filter by rating, date, and performance metrics.

📊 Compare

Side-by-side benchmarks: accuracy, F1 score, response time, model size, context window.

🚀 Share

Upload your model with a GitHub link, tags, and benchmarks. Get downloads and reviews.

🏆 Get Recognised

Appear on the leaderboard. Build your reputation as an AI engineer in your domain.

What is a Small Language Model?

An SLM is an AI language model fine-tuned for a specific task, typically under 7B parameters. They run on consumer hardware, cost a fraction of GPT-4 API calls, and outperform large general models on their target task because of domain-specific training.

👤 Create an Account

You can browse and download models without an account. You need an account to upload models, write reviews, save bookmarks, and appear on the leaderboard.

Sign up with email

1
Go to the auth page
Click Sign Up in the top right corner, or go to /auth.
2
Fill in your details
Enter your full name, email address, and a password (minimum 8 characters). The password strength indicator shows how secure your password is.
3
Confirm your email
Check your inbox for a confirmation email from SLM Marketplace. Click the link inside. The link is valid for 24 hours.
4
You're in
You'll be redirected to the homepage. Your initial letter appears in the top right header confirming you're logged in.

Sign up with GitHub

Click Continue with GitHub on the auth page. You'll be redirected to GitHub to authorise the app. Your GitHub username, name, and profile picture are imported automatically. No email confirmation needed.

ℹ️ GitHub sign-in is the fastest option for engineers — it connects your existing GitHub profile directly to your SLM Marketplace account.

Forgot your password?

On the login page, enter your email address in the email field, then click Forgot password?. A reset link will be emailed to you. The link expires after 1 hour.

⬇️ Downloading a Model

1
Click "Get Model"
On the model detail page, click the orange Get Model button. This opens the model's GitHub repository in a new tab.
2
Go to the GitHub repository
Read the README for installation and usage instructions. Each model has different requirements.
3
Download from HuggingFace or GitHub
Most models link to a HuggingFace model page where you can download weights directly. Use huggingface-hub or git clone.
⚠️ Check the license before using a model in production. Apache 2.0 and MIT allow commercial use. CC BY-NC does not. The license is shown on the Overview tab.

Running a model locally

Most models on SLM Marketplace can be run with the transformers library:

pip install transformers torch

from transformers import pipeline
pipe = pipeline("text-generation", model="username/ModelName")
result = pipe("Your input text here", max_new_tokens=256)

Check the model's GitHub README for model-specific instructions and any additional dependencies.

📊 Understanding Benchmarks

MetricWhat it measuresGood range
Accuracy% of predictions exactly correct on the test set85%+ for domain tasks
F1 ScoreBalance of precision and recall — more robust than accuracy alone80%+ good, 90%+ excellent
Response TimeAverage milliseconds to generate output (CPU, 512 tokens)<200ms ideal, <500ms acceptable
Model SizeNumber of parameters (1B, 3B, 7B)Smaller = faster/cheaper
Context WindowMaximum input tokens the model can process at once4K minimum, 8K+ preferred
QuantizedWhether weights are compressed (4-bit/8-bit) to save memoryYes = smaller, faster (slight quality trade-off)
ℹ️ Benchmarks are self-reported by model authors. Always test a model on your own data before deploying to production. A high accuracy score on the training benchmark may not transfer to your specific use case.

How to compare two models

Open each model's Benchmarks tab side by side. Prioritise F1 Score over Accuracy if your task has imbalanced classes. Prioritise Response Time if you need real-time inference. Prioritise Context Window if your inputs are long documents.

⭐ Writing Reviews

Reviews help the community understand how a model performs in real-world use — not just on benchmark tests. Good reviews are specific, honest, and mention your use case.

1
Use the model first
Test the model on your actual data or use case. Reviews based on real usage are far more valuable than reviews based on the benchmark numbers alone.
2
Go to the Reviews tab
On the model page, click the Reviews tab. You must be signed in. Each user can leave one review per model.
3
Select a star rating
Click the stars to rate (1–5). 5 stars = excellent, meets or exceeds benchmarks in real use. 3 stars = decent, some limitations. 1–2 stars = significant issues.
4
Write your comment
Describe your use case, what worked, what didn't. Mention specific inputs or scenarios. Keep it constructive.
✅ Great review example: "Used this for contract clause extraction in SaaS agreements. F1 on our internal test set was 61% (slightly below the listed 72%). Excellent on Limitation of Liability clauses, weaker on non-standard IP ownership terms. Overall recommended for standard commercial contracts."

🚀 Uploading a Model

Sharing your model takes about 5 minutes. You need a GitHub repository with the model files or HuggingFace link before you start.

1
Sign in and go to Upload
Click Upload Model in the navigation. You must be signed in — the page redirects to login if you're not.
2
Fill in the model name
Use a clear, descriptive name. Convention: TaskName-SLM-Size e.g. Legal-SLM-3B or MedCoder-SLM-1B. No spaces — use hyphens.
3
Select a category
Choose the primary domain. If your model spans multiple categories, pick the most dominant one and add the others as tags.
4
Write a short description
Max 150 characters. This appears in search results and model cards. Lead with what the model does, not how it was built. Example: "Assigns ICD-10-CM and CPT codes from clinical notes with 69% exact match accuracy."
5
Add your GitHub URL
Must be a valid public GitHub repository (e.g. https://github.com/username/repo). This is where users go to download the model.
6
Add tags
Type a tag and press Enter. Add 3–8 tags. Use the suggested tags that appear after selecting a category. Good tags: base model name, task type, techniques, language.
7
Click Continue
Your model is published immediately. You'll be redirected to its detail page. You can edit or delete the model from that page if you're the owner.
💡 Save Draft — click this instead of Continue to save without publishing. Drafts are visible only to you. Publish when your model and description are ready.

✅ Upload Guidelines

What's allowed

✅ Open-source models

Apache 2.0, MIT, CC BY, GPL, and similar permissive licenses.

✅ Fine-tuned models

Models fine-tuned from open base models (Llama, Mistral, Phi, Qwen, etc.).

✅ Honest benchmarks

Benchmarks run on real test sets. Include methodology in your GitHub README.

✅ Clear documentation

GitHub README with installation steps, usage example, and known limitations.

What's not allowed

🚫
Prohibited content:
Models trained on illegally obtained data · Models designed to deceive, harm, or generate illegal content · Benchmark fraud (inflated or fabricated numbers) · Duplicate submissions · Models with no working GitHub link · Closed-source models presented as open-source

GitHub repository requirements

Your repo must include at minimum:

  • A README.md with what the model does
  • Installation and usage instructions
  • The license file
  • How to download the model weights (HuggingFace link, direct download, or similar)

🏆 Leaderboard & Scoring

The leaderboard ranks engineers and models across three views. Filter by This Month, This Week, or All Time.

How the engineer score is calculated

Your score is computed automatically by the platform whenever your models get downloads or reviews:

Score = log₁₀(total downloads) × 100
      + (average rating / 5) × 300
      + min(model count × 20, 400)

Downloads carry the most weight. A model with 10,000 downloads scores much higher than one with 100, even if they have the same rating. Publishing more models increases your score up to a cap of 400 points.

🔖 Saving Models

Save any model to your bookmarks by clicking the bookmark icon on the top right of a model card (on Explore) or the bookmark icon on the model detail page. You must be signed in.

Your saved models appear in the Saved section on your profile page. Only you can see your saved models — they are private.

ℹ️ Saving a model does not download it. It just adds it to your personal list for quick access later.

❓ FAQ

Is SLM Marketplace free to use?
Yes, completely free. Browsing, downloading, and uploading models are all free. There are no plans to charge for core discovery features.
Can I upload a model without a GitHub repository?
The GitHub URL field is optional in the form, but we strongly recommend it. Without a repository link, users have no way to download or inspect your model, which dramatically reduces your downloads and credibility.
Can I upload a model I fine-tuned from Llama / Mistral / Phi?
Yes, as long as your fine-tune complies with the base model's license. Llama models require the Meta Llama Community License. Mistral and Phi use Apache 2.0. Always declare the base model and your fine-tuning license clearly in your README.
How do I get the verified checkmark?
Verified status is granted to engineers who have published multiple high-quality models with honest benchmarks, and whose models have received positive community reviews. It is manually reviewed. Contact support if you believe you qualify.
Can I edit or delete my model after publishing?
Yes. Go to your model's detail page. If you are the owner, you will see Edit and Delete buttons next to the Get Model button. Edit opens the upload form pre-filled with your model's data. Delete requires confirmation and is permanent.
My confirmation email never arrived. What do I do?
Check your spam/junk folder first. The email comes from noreply@sannan.app. If it's not there after 5 minutes, try signing up again — you can request a new confirmation email. If the problem persists, contact support.
Are the benchmarks verified by SLM Marketplace?
Benchmarks are self-reported by model authors and are not independently verified. Community reviews are the best signal of real-world performance. If you notice inflated or fraudulent benchmarks, use the Report button on the model page.
Can I use models from here in my commercial product?
It depends on the model's license — always check the License field on the Overview tab. Apache 2.0 and MIT permit commercial use. CC BY-NC does not. GPL has copyleft requirements. SLM Marketplace itself does not grant any rights — the license from the model author governs usage.
