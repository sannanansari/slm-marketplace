⚡ Foundations

LoRA

Understand LoRA from first principles: low-rank matrix decomposition, why it works, the r and alpha hyperparameters, and how QLoRA extends it.

The Problem LoRA Solves

Full fine-tuning updates every parameter in a model. A 7B model has 7 billion floats — in fp16 that is 14 GB of gradients alone, before optimizer states. That requires an A100 cluster. LoRA makes fine-tuning possible on a single consumer GPU.

How LoRA Works

LoRA observes that the weight updates during fine-tuning are low-rank. Instead of updating W directly, it learns two small matrices A (d×r) and B (r×d) where r ≪ d. The effective update is ΔW = B × A.

Full fine-tune: update W  (4096×4096 = 16.7M params per layer)
LoRA:           learn A (4096×16) and B (16×4096) = 131K params
                Reduction: 128× fewer trainable parameters

W_output = W_frozen + (B @ A) × (alpha / r)
           ↑ base model   ↑ LoRA adapter (your domain)
python
from peft import LoraConfig, get_peft_model
from transformers import AutoModelForCausalLM

model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-3.2-1B-Instruct")

lora_config = LoraConfig(
    r=16,           # rank — higher = more capacity
    lora_alpha=32,  # scale factor (usually 2×r)
    lora_dropout=0.05,
    target_modules=["q_proj", "k_proj", "v_proj", "o_proj"],
    bias="none",
    task_type="CAUSAL_LM"
)

model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# trainable params: 3,276,800 || all params: 1,238,710,272 || 0.26%

r and alpha Guide

Rank (r)Use whenVRAM impact
4–8Simple classification, style transferMinimal
16Most instruction-following tasks ✅ defaultLow
32–64Complex reasoning, diverse datasetsModerate
128+Approaching full fine-tune qualityHigh
