Transformer Architecture Explained

How transformer models work: self-attention, multi-head attention, feed-forward layers, positional encoding — with diagrams and intuition.

Why Transformers Changed Everything

Before transformers (2017), language models used RNNs which processed text sequentially — slow, and unable to capture long-range dependencies. The transformer processes all tokens in parallel using attention, making it massively faster and better at understanding context across long sequences.

The Architecture

Input Tokens: ["The", "cat", "sat"]

Self-Attention Intuition

For each token, attention computes three vectors: Query (what am I looking for?), Key (what do I contain?), Value (what do I contribute?). The attention score between two tokens = dot product of Query × Key, scaled and softmax-normalised.

python
import torch, torch.nn.functional as F

def self_attention(Q, K, V):
    d_k = Q.size(-1)  # dimension of keys
    # Score: how much each token attends to every other
    scores = torch.matmul(Q, K.transpose(-2, -1)) / d_k**0.5
    # Softmax: convert to probabilities
    weights = F.softmax(scores, dim=-1)
    # Weighted sum of Values
    return torch.matmul(weights, V)
