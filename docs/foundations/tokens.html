Tokenization

How BPE, WordPiece and SentencePiece tokenizers convert text to token IDs. Why tokenization affects model cost, quality, and multilingual performance.

What Is Tokenization?

A language model cannot read text. It reads numbers. Tokenization is the process of converting text into a sequence of integers (token IDs) that the model can process. The vocabulary — the set of all possible tokens — is fixed at training time.

BPE — Byte-Pair Encoding

The most common algorithm. Starts with characters, then iteratively merges the most frequent adjacent pairs. "playing" might be split as ["play", "ing"] → [1234, 5678].

python
from transformers import AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-3.2-1B")

text = "Fine-tuning a small language model"
tokens = tokenizer.tokenize(text)
ids    = tokenizer.encode(text)

print("Tokens:", tokens)
# ['Fine', '-', 'tuning', '▁a', '▁small', '▁language', '▁model']
print("IDs:", ids)
# [28724, 28733, 28707, 264, 2613, 4028, 2229]
print("Token count:", len(ids))  # 7

# Token count matters for cost and context window!
# GPT-4 pricing is per token, not per word.

Practical Implications

Why token count matters:
  • API costs are per token — longer tokenizations = higher cost
  • Context window is in tokens — 4096 tokens ≠ 4096 words
  • Non-English text often tokenizes less efficiently (uses more tokens per word)
  • Code tokenizes differently than prose — choose a code-optimised tokenizer for code tasks
