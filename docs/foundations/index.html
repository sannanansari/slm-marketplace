Transformers in 60 Seconds

Every SLM is built on the Transformer architecture. A Transformer reads all tokens in your input simultaneously and uses attention to weigh how important each token is relative to every other token when producing the next word.

🎯
Mental model

Think of it as: for every word being generated, the model asks “which other words in my input are most relevant right now?” — and weights their influence accordingly.

Tokens and Context Windows

Models don't read words — they read tokens (~0.75 words each in English). “Hello world” = 2 tokens. A 1,000-word document ≈ 1,300 tokens.

The context window is the maximum tokens a model processes at once. Modern SLMs range from 4K to 128K tokens.

Context WindowApprox WordsUse For
4K tokens~3,000Short documents, classification
8K tokens~6,000Medical notes, short contracts
32K tokens~24,000Long legal documents
128K tokens~96,000Full codebases, research papers

LoRA & QLoRA

LoRA adds small adapter matrices to specific transformer layers and only trains those — instead of all 7B weights. Result: fine-tune a 7B model with 8GB VRAM instead of 80GB, 3–5x faster, within 1–2% of full fine-tuning quality.

QLoRA first quantizes the model to 4-bit precision, then applies LoRA. A 7B model fits in 6GB VRAM.
