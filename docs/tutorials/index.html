Build Your First SLM

From zero to a published model on SLM Marketplace. Estimated time: 4–6 hours on free Colab.

⏱ 4–6 hoursIntermediate💰 Free (Colab T4)

What You Will Build

A domain-specific 1B parameter model fine-tuned on medical coding. By the end you will have:

  • A trained QLoRA model with benchmarks (accuracy, F1)
  • A quantized GGUF version running in Ollama
  • A REST API serving predictions
  • A published listing on SLM Marketplace
ℹ️
You can swap the domain

This tutorial uses medical coding as the example, but the exact same steps apply to any domain: legal, finance, coding, customer support. Just swap the dataset and system prompt.

Step 1 — Set Up Colab

Open Google Colab → Runtime → Change runtime type → T4 GPU.

Step 2 — Load Base Model

python
from unsloth import FastLanguageModel
import torch

model, tokenizer = FastLanguageModel.from_pretrained(
    model_name='unsloth/Llama-3.2-1B-Instruct',  # 1B = trains on free T4
    max_seq_length=1024,
    load_in_4bit=True,      # QLoRA
    dtype=torch.bfloat16,
)

# Add LoRA adapters (only these layers are trained)
model = FastLanguageModel.get_peft_model(
    model,
    r=16,                   # LoRA rank
    lora_alpha=32,
    lora_dropout=0.05,
    target_modules=['q_proj','k_proj','v_proj','o_proj','gate_proj','up_proj','down_proj'],
)
model.print_trainable_parameters()
# trainable params: 8,388,608 || all params: 1,244,739,584 (0.67%)

Step 3 — Prepare Dataset

python
from datasets import Dataset
import json

# Instruction-following format
examples = [
    {
        'instruction': 'Assign ICD-10-CM codes to this clinical note.',
        'input': 'Patient presents with acute chest pain, ST elevation V1-V4, troponin elevated.',
        'output': 'I21.9 - Acute myocardial infarction, unspecified'
    },
    # ... your full dataset here
]

SYSTEM_PROMPT = 'You are a medical coding assistant. Given a clinical note, assign the correct ICD-10-CM codes.'

def format_example(ex):
    return {'text': (
        f'<|system|>{SYSTEM_PROMPT}<|end|>'
        f'<|user|>{ex["instruction"]}\n{ex["input"]}<|end|>'
        f'<|assistant|>{ex["output"]}<|end|>'
    )}

dataset = Dataset.from_list(examples).map(format_example)
train_ds, eval_ds = dataset.train_test_split(test_size=0.15).values()
print(f'Train: {len(train_ds)} | Eval: {len(eval_ds)}')

Step 4 — Train

python
from trl import SFTTrainer
from transformers import TrainingArguments

trainer = SFTTrainer(
    model=model,
    tokenizer=tokenizer,
    train_dataset=train_ds,
    eval_dataset=eval_ds,
    dataset_text_field='text',
    max_seq_length=1024,
    args=TrainingArguments(
        output_dir='./medcoder-slm',
        num_train_epochs=3,
        per_device_train_batch_size=4,
        gradient_accumulation_steps=4,     # effective batch = 16
        learning_rate=2e-4,
        warmup_ratio=0.05,
        lr_scheduler_type='cosine',
        optim='paged_adamw_8bit',          # memory-efficient optimizer
        bf16=True,
        logging_steps=10,
        eval_strategy='steps',
        eval_steps=50,
        save_strategy='steps',
        save_steps=50,
        load_best_model_at_end=True,
        report_to='none',
    ),
)

trainer.train()
# Training time: ~4 hours on T4, ~1.5 hours on A100

Step 5 — Evaluate

python
from sklearn.metrics import accuracy_score, f1_score

def predict(note):
    inputs = tokenizer(note, return_tensors='pt').to(model.device)
    with torch.no_grad():
        outputs = model.generate(**inputs, max_new_tokens=128, temperature=0.1)
    return tokenizer.decode(outputs[0][inputs['input_ids'].shape[1]:], skip_special_tokens=True)

# Run on eval set
y_true, y_pred = [], []
for ex in eval_ds.select(range(200)):
    pred = predict(ex['input'])
    y_true.append(ex['output'].split()[0])  # first ICD code
    y_pred.append(pred.split()[0])

print(f'Exact Match: {accuracy_score(y_true, y_pred):.1%}')
print(f'F1 (macro):  {f1_score(y_true, y_pred, average="macro", zero_division=0):.1%}')

Step 6 — Quantize to GGUF

python
# Save as GGUF Q4_K_M (4x smaller, ~3% quality loss)
model.save_pretrained_gguf(
    'medcoder-gguf',
    tokenizer,
    quantization_method='q4_k_m',  # recommended: small + good quality
)
# Creates: medcoder-gguf/unsloth.Q4_K_M.gguf (~500MB for 1B model)

Step 7 — Push to HuggingFace

python
from huggingface_hub import HfApi, login

login()  # paste your HF write token

# Push merged model
model.push_to_hub('your-username/MedCoder-SLM-1B', tokenizer=tokenizer)

# Push GGUF version
api = HfApi()
api.upload_folder(
    folder_path='medcoder-gguf',
    repo_id='your-username/MedCoder-SLM-1B-GGUF',
    repo_type='model'
)
print('Model live at: https://huggingface.co/your-username/MedCoder-SLM-1B')

Step 8 — Run Locally with Ollama

bash
# Download your model
ollama pull your-username/medcoder-slm-1b

# Or create from local GGUF
cat > Modelfile << 'EOF'
FROM ./medcoder-gguf/unsloth.Q4_K_M.gguf
SYSTEM 'You are a medical coding assistant. Assign ICD-10-CM codes to clinical notes.'
EOF

ollama create medcoder -f Modelfile
ollama run medcoder 'Patient with acute chest pain, ST elevation in V1-V4...'

3
Enter your benchmark numbers
Accuracy: your exact match %. F1: your F1 score. Response time: ~280ms on T4.
4
Publish → Share!
Your model is live. Share on r/LocalLLaMA and LinkedIn. First downloads incoming.
